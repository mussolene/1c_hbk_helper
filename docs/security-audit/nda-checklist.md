# Чеклист NDA: аудит безопасности проекта 1C Help MCP

Использовать при проведении аудита в условиях конфиденциальности и NDA.

## Перед началом аудита

- [ ] Подписано соглашение о неразглашении (NDA) между заказчиком и исполнителем аудита
- [ ] Определён перечень лиц, имеющих право доступа к результатам аудита
- [ ] Выделено защищённое хранилище для артефактов аудита (отчёты, логи, скриншоты)
- [ ] Установлены правила передачи отчёта: только уполномоченным лицам, по согласованию
- [ ] Согласована политика хранения: срок, способ удаления по окончании

## В процессе аудита

- [ ] Код и данные обрабатываются только в защищённой среде (изолированная машина / VPN)
- [ ] Логи и временные файлы не содержат конфиденциальных путей, токенов, фрагментов кода
- [ ] Скриншоты и воспроизведения уязвимостей обезличены (пути, имена, данные)
- [ ] Обмен с заказчиком — через защищённые каналы (шифрование, доверенная почта)
- [ ] Исполнитель не публикует и не передаёт третьим лицам код, данные и находки

## После завершения аудита

- [ ] Отчёт и находки сохранены в защищённом хранилище с контролем доступа
- [ ] Передача отчёта только уполномоченным лицам по согласованию с заказчиком
- [ ] Удалены временные копии репозитория и артефактов с рабочих машин исполнителя
- [ ] Публикация отчёта или результатов — только после письменного снятия ограничений
- [ ] Срок соблюдения NDA и условия досрочного прекращения зафиксированы в соглашении

## При работе с конфиденциальным кодом 1С

- [ ] Сниппеты и примеры кода в отчёте — обезличены или заменены на синтетические
- [ ] SNIPPETS_DIR, MEMORY_BASE_PATH, HELP_SOURCE_BASE — не раскрываются в отчёте
- [ ] EMBEDDING_API_URL и токены — никогда не включаются в отчёт или логи
- [ ] При NDA заказчика: embedding API и память — только on-prem; внешние сервисы не используются для конфиденциальных данных
